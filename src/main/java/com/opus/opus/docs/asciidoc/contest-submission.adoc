== 제출물

=== `GET`: 제출물 상세 조회

한 팀이 특정 제출 항목로 낸 제출 한 건의 상세 정보를 조회합니다.
`ROLE_학생`은 해당 팀의 팀장/팀원만 조회할 수 있으며, 관리자/교수/직원/외부멘토는 제한이 없습니다.
`status`는 계산값으로, 단건 조회에서는 `SUBMITTED`(마감 내) / `LATE`(마감 후)만 반환됩니다.
`commentCount`는 제출물 코멘트 기능 추가 전까지 항상 0입니다.

=== `GET`: 팀 제출물 현황 목록 조회

팀 페이지에서 해당 팀이 제출해야 하는 제출물 항목 전체와, 항목별 제출 현황을 조회합니다.
`ROLE_학생`은 해당 팀의 팀장/팀원만 조회할 수 있으며, 관리자는 제한이 없습니다.
제출 상태(`status`)는 저장값이 아니라 마감일시와 실제 제출 여부를 조회 시점에 비교하여 계산하며, `status` 파라미터로 필터링할 수 있습니다.
미제출 항목은 `submissionId`가 `null`이고 `files`는 빈 배열입니다.

=== `GET`: 마감 예정 제출 항목 목록 조회

특정 팀의 마감일이 지나지 않은(`deadlineAt`이 조회 시점 이후인) 제출 항목만 마감일 오름차순으로 조회합니다.
`ROLE_학생`은 해당 팀의 팀장/팀원만 조회할 수 있으며, 관리자는 제한이 없습니다.
제출 상태(`status`)는 저장값이 아니라 마감일시와 실제 제출 여부를 조회 시점에 비교하여 계산하며, `SUBMITTED` / `LATE` / `NOT_SUBMITTED` 중 하나입니다.
미제출 항목은 `lastModifiedAt`이 `null`입니다.

=== `POST`: 제출물 제출

팀이 특정 제출 항목에 파일을 제출합니다. (`multipart/form-data`, 여러 파일 가능) 한 팀이 동일한 제출 항목에 중복 제출할 수 없으며, 파일을 추가하려면 "제출 파일 추가" API를 사용합니다.
`teamId`는 요청 파라미터로 전달하며, `ROLE_학생`은 해당 팀의 팀장/팀원이어야 합니다.
